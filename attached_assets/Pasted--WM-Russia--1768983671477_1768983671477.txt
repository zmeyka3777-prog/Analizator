Создай профессиональное веб-приложение "Анализатор продаж WM Russia" — специализированную платформу для управления и анализа продаж фармацевтической компании WM Russia с системой разграничения доступа по ролям, детальной аналитикой и визуализацией данных.

🏗️ ТЕХНИЧЕСКАЯ БАЗА
Стек технологий:

React 18+ с TypeScript
Tailwind CSS v4 для стилизации
Recharts для графиков и визуализации
Lucide React для иконок
Shadcn/ui для UI-компонентов
React Router для навигации (если необходимо)
Основной файл: /App.tsx с дефолтным экспортом

👥 СИСТЕМА РОЛЕЙ И ДОСТУПА
1. Медицинский представитель (medrep)
Доступ:

✅ Личная статистика продаж (только свои данные)
✅ План/факт по своим показателям (упаковки и деньги)
✅ Графики по своим продажам за период
✅ Сравнение своих результатов с прошлыми месяцами
✅ Личный рейтинг в команде округа (позиция среди коллег)
✅ Детализация продаж по каждому из 12 препаратов
❌ НЕ видит данные других медпредов
❌ НЕ видит общую статистику округа
❌ НЕ видит аналитику других регионов
Интерфейс для медпреда:

Дашборд с персональными KPI (план/факт по упаковкам и деньгам)
Круговая диаграмма распределения продаж по 12 препаратам
Линейный график динамики продаж за последние 6 месяцев
Таблица с детализацией: препарат, план, факт, % выполнения
Badge с позицией в рейтинге команды
Прогресс-бары выполнения плана по каждому продукту
2. Территориальный менеджер (territory_manager)
Доступ:

✅ Данные всех медпредов своей территории (город/район)
✅ Сравнительная таблица медпредов территории
✅ План/факт территории в агрегированном виде
✅ Рейтинг медпредов по выполнению плана
✅ Детализация по каждому медпреду (все 12 препаратов)
✅ Графики динамики продаж территории
❌ НЕ видит другие территории своего округа
❌ НЕ видит данные других округов
❌ НЕ имеет доступа к загрузке данных
Интерфейс для территориального менеджера:

Дашборд с общими показателями территории
Сравнительная таблица медпредов с сортировкой и фильтрами
Визуализация топ-3 и аутсайдеров по выполнению плана
Детальная карточка каждого медпреда (расширяемая)
Графики сравнения план/факт для территории
Возможность переключения между представлениями: карточки / таблица
3. Региональный менеджер (manager)
Доступ:

✅ Полная аналитика своего федерального округа (например, ПФО)
✅ Данные всех территорий округа
✅ Данные всех медпредов округа
✅ Сравнение территорий внутри округа
✅ План/факт округа по упаковкам и деньгам
✅ Детализация по всем 12 препаратам на уровне округа
✅ Рейтинг медпредов и территорий
✅ Графики динамики, тренды, прогнозы
❌ НЕ видит другие федеральные округа
❌ НЕ имеет доступа к загрузке данных
❌ НЕ видит админ-панель
Интерфейс для регионального менеджера:

Полноценная аналитическая панель округа
Сводная статистика: план/факт по деньгам и упаковкам
Круговая диаграмма распределения продаж по 12 препаратам
Сравнительная таблица всех медпредов округа
Группировка по территориям с возможностью разворачивания
Визуализация трендов и динамики
Фильтры по периодам, препаратам, территориям
Экспорт отчётов (PDF, Excel)
4. Директор компании (admin)
Полный доступ ко всему:

✅ Аналитика по всем 8 федеральным округам России

✅ Данные всех территорий и медпредов всех округов

✅ Полная админ-панель с расширенной аналитикой
✅ Журнал загрузок и изменений данных
✅ Экспорт любых отчётов и аналитики

// utils/calculations.ts

export function calculatePerformance(plan: number, fact: number): number {
  if (plan === 0) return 0;
  return Math.round((fact / plan) * 100 * 10) / 10;
}

export function getStatusByPerformance(performance: number): 'success' | 'warning' | 'danger' {
  if (performance >= 95) return 'success';
  if (performance >= 85) return 'warning';
  return 'danger';
}

export function aggregateMedRepsData(medReps: MedRepData[]): {
  totalPackagesPlan: number;
  totalPackagesFact: number;
  totalMoneyPlan: number;
  totalMoneyFact: number;
} {
  return medReps.reduce((acc, rep) => ({
    totalPackagesPlan: acc.totalPackagesPlan + rep.totalPackagesPlan,
    totalPackagesFact: acc.totalPackagesFact + rep.totalPackagesFact,
    totalMoneyPlan: acc.totalMoneyPlan + rep.totalMoneyPlan,
    totalMoneyFact: acc.totalMoneyFact + rep.totalMoneyFact
  }), {
    totalPackagesPlan: 0,
    totalPackagesFact: 0,
    totalMoneyPlan: 0,
    totalMoneyFact: 0
  });
}

export function sortMedRepsByPerformance(medReps: MedRepData[], order: 'asc' | 'desc' = 'desc'): MedRepData[] {
  const sorted = [...medReps].sort((a, b) => {
    const perfA = calculatePerformance(a.totalMoneyPlan, a.totalMoneyFact);
    const perfB = calculatePerformance(b.totalMoneyPlan, b.totalMoneyFact);
    return perfB - perfA;
  });
  return order === 'asc' ? sorted.reverse() : sorted;
}

// utils/formatters.ts

export function formatCurrency(amount: number): string {
  return new Intl.NumberFormat('ru-RU', {
    style: 'currency',
    currency: 'RUB',
    minimumFractionDigits: 0,
    maximumFractionDigits: 0
  }).format(amount);
}

export function formatNumber(num: number): string {
  return new Intl.NumberFormat('ru-RU').format(num);
}

export function formatPercentage(value: number): string {
  return `${value.toFixed(1)}%`;
}
