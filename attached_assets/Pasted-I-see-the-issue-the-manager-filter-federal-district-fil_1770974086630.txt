I fixed the critical filter bug. Here's what was happening and what I changed:

The Problem: When you selected a manager (like Оруджов Али) whose assigned regions (Moscow, etc.) didn't exist in the user's data, the system was supposed to show empty results but instead showed ALL unfiltered data — making it look like the filter wasn't working at all. Same thing happened when selecting regions that don't match any data.

The Root Cause: There was a "safety net" in the code designed for old/legacy data uploads that don't have detailed row records. When filtering produced 0 results for any year, this safety net would kick in and return all the original unfiltered data for that year — even when the 0 results were the correct answer (the selected region simply doesn't exist in the data).

The Fix: The safety net now only activates for years that truly have no detailed row records (legacy uploads). If a year has detailed records but filtering produces 0 matches, that's treated as a legitimate empty result — the filter is working correctly, there's just no data matching the criteria.

This fixes both the manager filter and region filter issues — they were both caused by the same underlying bug.
